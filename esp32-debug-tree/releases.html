リリースノート — 技術的修正・機能追加の履歴

v3.2 2026-03-22
技術的誤記の修正
1 TWDT / loopTaskWDTEnabled 誤記修正
誤loopTask は TWDT の監視対象にデフォルト登録されている（v2.x）
正v1.0.6 / v2.0.17 / v3.x の全バージョンで loopTaskWDTEnabled = false が明示セットされており、loopTask は TWDT 監視対象に自動登録されない
根拠v1.0.6・v2.0.17 の main.cpp ソースコードを直接確認
対応enableLoopWDT() による手動有効化の方法を追加
2 コアの割り当て（Core0 / Core1） 誤記修正
誤Core0 が Arduino loop、Core1 が WiFi/BT
正Core0 = PRO_CPU（WiFi/BT スタック）、Core1 = APP_CPU（Arduino loop のデフォルト）
根拠ARDUINO_RUNNING_CORE のデフォルト値は 1
3 ストラッピングピン本数 誤記修正
誤4本（GPIO0/2/12/15）
正5本（GPIO0/2/5/12/15）。GPIO5 が抜けていた
根拠ESP32 データシート p.21 / ESP-IDF 公式ドキュメント
対応各ピンの動作説明を詳細化、GPIO_STRAP レジスタのビット構成を追記
4 UART0 起動時ボーレート 誤記修正
誤ESP32 は 74880bps で起動ログを出力する
正ESP32 は 115200bps。74880bps は ESP8266（26MHz クリスタル）固有の値
根拠esptool 公式ドキュメント / Espressif フォーラム
5 ISR スタックサイズの明記 情報追加
追加CONFIG_FREERTOS_ISR_STACKSIZE のデフォルトは 1536 バイト / コア
追加コアダンプ有効時は 2096 バイト、ESP32 デュアルコアで合計 3072 バイト消費
根拠esp-idf/components/freertos/Kconfig を直接確認
6 ISR 禁止関数リストの拡充 情報追加
追加analogRead()（ADC Mutex を使用。ISR から呼ぶと例外発生の実績あり）
追加SPI.transfer、Wire.read/write、malloc/new、String 操作、WiFi/BT API
対応従来の Serial.print / delay に加えて Arduino API 全般を禁止として明記
診断内容の強化
7 SPI マルチデバイス競合（r_spi_multi） 内容強化
追加複数タスクから SPI バスを使う場合の Mutex 必須説明
追加速度切り替え忘れによるデータ化けのリスク説明
追加Mutex 取得 → beginTransaction → 通信 → endTransaction → Mutex 解放の正しい順序のサンプルコード
8 I2C クロック混在（r_i2c_clock） 内容強化
追加同一バスで異なる最大速度のデバイスを同時通信できない原理の説明
追加解決策 3 パターン（最低速度に統一 / Wire1 でバス分離 / TCA9548A でマルチプレクサ）
追加Standard Mode 100kHz / Fast Mode 400kHz / Fast Mode Plus 1MHz の規格説明
9 ISR スタックサイズ（r_isr_heavy） 内容強化
追加ISR 専用スタックのデフォルト 1536 バイトを cause / fix / info に明記
追加スタック溢れが通常の canary では検出できない点を警告
コード品質の改善
10 全 69 コードブロックを完全スケッチ化 コード品質
修正前setup() または loop() が欠落しているブロックが 64 件
修正後全 69 ブロックが setup() + loop() を含む、そのままコピーして動作するスケッチ
対応タスク関数は setup() 内で xTaskCreateUniversal() 起動するパターンに統一
UI 改善
11 ハードウェアチェックリストをドロップダウン化 UI
TOP診断カード内に展開表示（スクロール不要）
他ページ折りたたみバナーとして表示。ページ遷移のたびに折りたたみ状態にリセット
操作クリック / タップで開閉。アイコンで開閉状態を表示
